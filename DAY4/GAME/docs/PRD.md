# 테트리스(Tetris) PRD

## 1. 프로젝트 개요

- **한 줄 요약**: 7종 테트로미노를 쌓아 라인을 채우고 지우며 점수를 겨루는 클래식 낙하형 퍼즐 게임.
- **장르**: 낙하형 퍼즐(Falling Block Puzzle)
- **타겟 플랫폼**: 웹 브라우저 (데스크톱 우선, 키보드 입력 기반)
- **타겟 사용자층**: 캐주얼 퍼즐 게임을 즐기는 일반 사용자, 클래식 테트리스에 익숙한 사용자
- **핵심 컨셉**: 표준 테트리스 룰(7-bag 랜덤, 벽차기 회전, 소프트/하드 드롭)을 충실히 구현한 싱글플레이 웹 게임
- **차별점**: 화려한 신규 기능 대신, 표준 규칙을 정확하고 반응성 좋게 구현하는 데 집중한다. 부트캠프 학습 프로젝트로서 Phaser 3의 Scene/GameObject/입력 처리를 명확하게 보여주는 레퍼런스 구현을 목표로 한다.
- **핵심 재미(Core Fun)**: (1) 빈틈없이 블록을 쌓았을 때 라인이 한 번에 지워지는 손맛, (2) 낙하 속도가 점점 빨라지며 긴장감이 상승하는 리스크-보상 루프, (3) 회전/벽차기를 활용해 불리한 상황을 극복하는 순간의 성취감.

## 2. 핵심 기능 목록

### 2.1 테트로미노 생성 및 랜덤 배출 (7-bag)
- **설명**: I, O, T, S, Z, J, L 7종 테트로미노를 정의하고, 편향 없는 순서로 하나씩 배출한다.
- **동작 방식**: 7개 블록을 하나의 "bag"에 넣고 무작위로 셔플한 뒤 순서대로 배출한다. bag이 비면 다시 7개를 채워 셔플한다(같은 블록이 연속 6번 이상 나오지 않도록 보장).
- **필요한 데이터/상태**: `currentBag: TetrominoType[]`, `activePiece`, `nextPiece`
- **예외/엣지 케이스**: 새 블록 생성 위치(스폰 지점)에 이미 다른 블록이 쌓여 있으면 즉시 게임 오버 처리로 이어진다(§2.7 참고).

### 2.2 블록 이동 및 회전 (벽차기 포함)
- **설명**: 좌/우 이동, 회전(시계/반시계)을 지원하며, 회전 시 보드 경계나 다른 블록과 충돌하면 미리 정의된 오프셋만큼 위치를 보정하는 벽차기(Wall Kick)를 적용한다.
- **동작 방식**: 방향키 입력 → 이동/회전 가능 여부를 보드 상태와 대조해 검증 → 가능하면 반영, 회전이 불가능하면 SRS(Super Rotation System) 기준 오프셋 후보를 순서대로 시도 → 모두 실패하면 회전 취소.
- **필요한 데이터/상태**: `activePiece.position`, `activePiece.rotationState`, 보드 그리드(`board: (TetrominoType | null)[][]`)
- **예외/엣지 케이스**: 보드 좌우 경계, 바닥, 다른 고정 블록과의 충돌을 모두 검사한다. O 블록은 회전해도 형태가 바뀌지 않으므로 벽차기 로직에서 조기 반환한다.

### 2.3 소프트 드롭 / 하드 드롭
- **설명**: 소프트 드롭은 낙하 속도를 임시로 가속하고, 하드 드롭은 즉시 바닥(또는 장애물)까지 낙하시켜 고정한다.
- **동작 방식**: 소프트 드롭 키를 누르는 동안 낙하 간격을 단축; 하드 드롭 키 입력 시 충돌 검사로 최대 낙하 위치를 계산해 즉시 이동 후 고정(lock) 처리.
- **필요한 데이터/상태**: `dropInterval`(현재 레벨 기준), `isSoftDropping: boolean`
- **예외/엣지 케이스**: 하드 드롭 직후 락 딜레이 없이 즉시 고정 및 라인 클리어 판정으로 이어져야 한다(연타로 중복 고정되지 않도록 상태 잠금 처리).

### 2.4 라인 클리어 판정 및 점수 계산
- **설명**: 블록이 고정된 후 가로 한 줄이 모두 채워졌는지 검사하고, 채워진 줄을 제거하며 점수를 부여한다.
- **동작 방식**: 블록 고정(lock) 이벤트 발생 → 보드의 모든 행을 검사 → 가득 찬 행을 제거하고 위쪽 블록을 아래로 당김 → 동시에 지워진 줄 수에 따라 점수 계산(1줄/2줄/3줄/4줄 테트리스 각각 다른 배점, 레벨에 비례해 가중).
- **필요한 데이터/상태**: `score: number`, `linesCleared: number`, `level: number`
- **예외/엣지 케이스**: 한 번의 고정으로 여러 줄이 동시에 지워지는 경우(더블/트리플/테트리스)를 올바르게 판정하고 배점을 차등 적용한다.

### 2.5 레벨업 및 낙하 속도 증가
- **설명**: 누적 라인 클리어 수가 일정 기준을 넘으면 레벨이 오르고, 블록 낙하 속도가 빨라진다.
- **동작 방식**: 라인 클리어 시 누적 카운트 갱신 → 레벨업 기준(예: 10줄마다 1레벨) 도달 시 레벨 증가 → 레벨에 따른 낙하 간격(ms)을 공식 또는 테이블로 재계산.
- **필요한 데이터/상태**: `totalLinesCleared: number`, `level: number`, `dropInterval: number`
- **예외/엣지 케이스**: 최고 레벨 이후에는 낙하 속도가 과도하게 빨라져 플레이 불가능한 수준이 되지 않도록 최소 간격 하한선을 둔다.

### 2.6 다음 블록 미리보기 (Next)
- **설명**: 현재 조작 중인 블록 다음에 나올 블록을 화면에 미리 표시한다.
- **동작 방식**: 7-bag에서 다음 순번 블록을 `nextPiece`로 유지하고, 별도 미리보기 영역에 렌더링. 현재 블록이 고정되면 `nextPiece`가 `activePiece`로 승격되고 bag에서 새 `nextPiece`를 채운다.
- **필요한 데이터/상태**: `nextPiece: TetrominoType`
- **예외/엣지 케이스**: bag이 비어 셔플이 필요한 시점에도 미리보기가 끊기지 않도록 항상 1개 이상 앞서 채워둔다.

### 2.7 게임 오버 조건 및 처리
- **설명**: 새로 생성된 블록이 스폰 위치에서 즉시 다른 블록과 충돌하면 게임을 종료한다.
- **동작 방식**: 새 블록 생성 시 스폰 위치 충돌 검사 → 충돌 시 게임 루프 정지, 게임 오버 화면(Scene) 전환, 최종 점수 표시 및 최고 점수 갱신 여부 확인.
- **필요한 데이터/상태**: `isGameOver: boolean`, `score`, `highScore`
- **예외/엣지 케이스**: 게임 오버 직후 남아있는 입력 이벤트가 처리되지 않도록 입력 리스너를 비활성화한다.

### 2.8 일시정지
- **설명**: 플레이 중 게임 진행을 멈추고 재개할 수 있다.
- **동작 방식**: 일시정지 키 입력 → 게임 루프(낙하 타이머, 입력 처리) 정지 및 일시정지 오버레이 표시 → 재입력 시 정확히 이전 상태에서 재개.
- **필요한 데이터/상태**: `isPaused: boolean`
- **예외/엣지 케이스**: 일시정지 중에는 이동/회전/드롭 입력이 모두 무시되어야 하며, 낙하 타이머도 함께 멈춰야 한다(재개 시 남은 시간부터 이어지도록 처리).

### 2.9 최고 점수 저장 (localStorage)
- **설명**: 게임 오버 시점의 점수를 이전 최고 점수와 비교해 갱신하고, 브라우저를 새로고침해도 유지되도록 저장한다.
- **동작 방식**: 게임 오버 시 `localStorage`에서 기존 최고 점수를 읽어 현재 점수와 비교 → 더 높으면 갱신 후 저장.
- **필요한 데이터/상태**: `localStorage['tetris_high_score']`
- **예외/엣지 케이스**: `localStorage` 접근이 차단된 환경(예: 프라이빗 브라우징 일부 설정)에서도 게임 자체는 정상 동작해야 하며, 저장 실패는 조용히 무시한다.

## 3. 사용자 시나리오

### 3.1 신규 유저 시나리오
1. 사용자가 게임 페이지에 접속하면 타이틀 화면에서 "시작" 버튼과 조작키 안내(← → 이동, ↑ 회전, ↓ 소프트드롭, Space 하드드롭, P 일시정지)를 확인한다.
2. "시작"을 누르면 빈 보드와 함께 첫 블록이 상단에서 낙하하기 시작하고, 우측에는 Next 블록과 현재 점수(0), 레벨(1)이 표시된다.
3. 사용자가 방향키로 블록을 이동/회전시키다가 실수로 회전이 막히는 상황(벽 근처)에 부딪히지만, 벽차기가 적용되어 자연스럽게 회전이 성공하는 것을 경험한다.
4. 첫 줄을 채워 라인이 지워지고 점수가 올라가는 것을 보며 게임 규칙을 익힌다.
5. 여러 줄을 놓쳐 블록이 쌓이고 결국 스폰 위치까지 도달해 게임 오버 화면이 뜬다. 최종 점수와 "최고 점수 달성!" 여부가 표시된다.
6. "다시 시작" 버튼으로 즉시 재도전한다.

### 3.2 반복 플레이 유저 시나리오
1. 기존 플레이어가 페이지에 재접속하면 타이틀 화면에 저장된 최고 점수가 표시된다.
2. "시작"을 눌러 바로 게임에 진입하고, 하드 드롭을 적극 활용해 빠른 속도로 블록을 쌓는다.
3. 레벨이 5까지 올라가며 낙하 속도가 눈에 띄게 빨라지자, 소프트 드롭과 빠른 회전 조작으로 대응한다.
4. 4줄을 동시에 지우는 "테트리스"를 달성해 큰 점수를 획득하고, 화면에 이를 인지할 수 있는 피드백(점수 급상승)을 확인한다.
5. 중간에 급한 일이 생겨 P키로 일시정지하고, 잠시 후 복귀해 정확히 같은 상태에서 재개한다.
6. 최종적으로 이전 최고 점수를 경신하고, 새로운 최고 점수가 `localStorage`에 저장되어 다음 방문 시에도 유지된다.

## 4. 기술 스택

- **언어**: TypeScript
- **게임 프레임워크**: Phaser 3 — Scene 전환(타이틀/플레이/게임오버), 입력 처리, 렌더링, 타이머(낙하 인터벌)에 Phaser의 내장 기능을 사용한다.
- **빌드 도구**: Vite
- **데이터 저장**: `localStorage` (최고 점수만 저장하므로 별도 서버/DB 불필요)
- **배포 환경**: 정적 웹 호스팅 (Vite 빌드 결과물을 그대로 배포)
- **구조 (CLAUDE.md 기준)**:
  - `scenes/`: `TitleScene`, `MainScene`(실제 플레이), `GameOverScene`
  - `objects/` 또는 `logic/`: 보드 상태, 테트로미노 정의, 충돌/회전/라인클리어 판정 로직 (순수 TypeScript로 작성해 테스트 용이성 확보)
  - `config/`: 보드 크기, 낙하 속도 테이블, 점수 배점 등 상수

## 5. 개발 우선순위

### MVP (필수, 1차 개발)
- 보드/그리드 렌더링, 테트로미노 7종 정의 및 7-bag 배출
- 블록 이동 · 회전(기본 회전, 벽차기는 단순 버전) · 소프트/하드 드롭
- 블록 고정(lock) 및 라인 클리어 판정, 기본 점수 계산
- 게임 오버 조건 및 게임 오버 화면
- Next 블록 미리보기
- **이유**: 위 기능이 없으면 "테트리스"로서 플레이가 불가능하다. 핵심 재미 루프(쌓기 → 클리어 → 반복)를 가장 먼저 완성해야 한다.
- **난이도**: 중 (충돌 판정과 회전 로직이 가장 까다로운 부분)

### 2차 기능
- 정식 SRS 벽차기 오프셋 테이블 적용 (MVP의 단순 벽차기를 정교화)
- 레벨업 시스템 및 레벨별 낙하 속도 증가
- 일시정지 기능
- 최고 점수 저장(localStorage) 및 타이틀 화면 표시
- **이유**: MVP의 플레이 루프가 검증된 뒤, 게임의 긴장감(난이도 상승)과 재방문 동기(최고 점수)를 강화하는 단계다.
- **난이도**: 하~중

### 추가 기능 (Nice to have)
- 홀드(Hold) 기능 (현재 블록을 보류하고 다음 블록 사용)
- 고스트 피스(낙하 예상 위치 미리보기)
- 사운드 이펙트/배경음악
- 키 입력 커스터마이징
- **이유**: 플레이 경험을 향상시키지만 핵심 재미 루프에는 필수가 아니므로 여유가 있을 때 추가한다.
- **난이도**: 하
