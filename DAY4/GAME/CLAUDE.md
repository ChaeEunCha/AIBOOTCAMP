# CLAUDE.md

이 문서는 이 프로젝트(GAME)에서 코드를 작성할 때 반드시 참조해야 하는 개발 가이드다. 프로젝트 소개나 기능 설명이 아니라, 실제 개발 작업 시 지켜야 할 규칙을 담고 있다.

## 기술 스택

- **언어**: TypeScript (모든 게임 로직/코드는 `.ts`로 작성하고, `any` 사용을 지양한다)
- **게임 프레임워크**: Phaser 3. 렌더링, 게임 루프, 입력, 물리, 씬 전환은 Phaser가 제공하는 기능을 사용하고 직접 재구현하지 않는다.
- **런타임 대상**: 브라우저
- **빌드/실행**: Vite + TypeScript로 개발/빌드한다 (별도 결정이 없다면 기본값).
- **데이터 저장**: 별도 백엔드가 필요 없는 범위에서는 `localStorage`를 사용한다. 서버/DB가 필요한 기능이 생기면 이 문서에 스택을 추가하고 결정 근거를 남긴다.
- **의존성**: Phaser 외 새로운 라이브러리를 추가하기 전에 반드시 필요성을 확인한다. Phaser 3가 기본 제공하는 기능(물리, 파티클, 트윈, 오디오 등)은 별도 라이브러리로 대체하지 않는다.

## 코드 스타일

- **타입**: 모든 함수의 파라미터/반환값에 타입을 명시한다. `any`는 금지하고, 불가피한 경우 `unknown` + 타입 가드로 처리한다. Phaser 타입 정의(`Phaser.Types.*`, `Phaser.GameObjects.*` 등)를 적극 활용한다.
- **네이밍**: 변수/함수는 `camelCase`, 타입/인터페이스/클래스/Scene은 `PascalCase`, 상수는 `UPPER_SNAKE_CASE`. Scene 클래스명은 역할이 드러나도록 짓는다 (예: `BootScene`, `TitleScene`, `MainScene`, `GameOverScene`).
- **구조**: Scene 단위로 파일을 분리한다 (`scenes/` 디렉토리). 게임 오브젝트(플레이어, 적, 아이템 등)는 `Phaser.GameObjects.GameObject`를 확장한 클래스로 `objects/` 디렉토리에 분리한다. 공용 설정값(화면 크기, 물리 상수 등)은 `config/` 디렉토리에 모은다.
- **상태 관리**: Scene 간 공유해야 하는 상태(점수, 스테이지 진행도 등)는 Phaser의 `registry` 또는 별도 상태 관리 모듈로 관리하고, 여러 Scene에서 같은 값을 중복으로 들고 있지 않는다.
- **주석**: 코드가 스스로 설명하지 못하는 부분(예: 특정 상수값의 유래, 물리/충돌 계산의 이유, Phaser API의 특이한 동작 우회)에만 짧게 작성한다. "무엇을 하는지"를 설명하는 주석은 쓰지 않는다.
- **포맷**: 들여쓰기 2 spaces, 세미콜론 사용, 문자열은 기본적으로 `'single quote'`를 사용한다.

## 개발 원칙

- **Phaser 생명주기 준수**: `preload`(에셋 로드) → `create`(초기화) → `update`(매 프레임 로직) 순서를 지키고, 에셋 로드와 게임 오브젝트 생성을 섞지 않는다.
- **Scene 분리**: 하나의 Scene이 여러 화면/상태의 책임을 갖지 않도록 한다. 화면 전환은 `this.scene.start()` 등 Phaser의 Scene 전환 API를 사용한다.
- **작게 쪼개서 검증**: 하나의 기능을 구현할 때 플레이 가능한 최소 단위로 나누고, 매 단계마다 `npm run dev`로 브라우저에서 직접 실행해 동작을 확인한다. 타입 체크가 통과했다고 기능이 맞다고 간주하지 않는다.
- **PRD와의 일치**: `.claude/agents/game-prd-writer.md`로 작성된 PRD가 있다면, 그 문서의 MVP 우선순위를 따라 개발 순서를 정한다. PRD와 다른 순서로 개발해야 할 이유가 생기면 먼저 PRD를 갱신한다.
- **불필요한 확장 금지**: 요청받은 기능 범위를 넘어서는 시스템(예: 요청하지 않은 멀티플레이, 과도한 설정 옵션)을 미리 만들지 않는다. 필요해질 때 추가한다.
- **성능**: `update` 루프 내부에서는 불필요한 객체 생성, DOM 접근, 콘솔 로그를 피한다. 반복 생성되는 오브젝트(총알, 이펙트 등)는 Phaser의 오브젝트 풀(`Phaser.GameObjects.Group`)을 사용한다.
- **엣지 케이스**: 입력값 경계(예: 체력 0 이하, 화면 밖으로 나가는 오브젝트, 빠른 연속 입력)를 항상 명시적으로 처리한다.
